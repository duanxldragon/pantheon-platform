# 前端 UI 前后对照模板

> 本文用于给 Pantheon Platform 的 UI 改造补正式对照材料。  
> 它回答：一页 UI 改造完成后，应该如何写“改造前 / 改造后 / 截图占位 / 验收说明”。  
> 交付口径看 `./UI_DELIVERY_CHECKLIST.md`，页面模板看 `./UI_PAGE_TEMPLATES.md`。

## 1. 使用方式

适合以下场景：

- 需要给团队同步某个页面这次到底改了什么；
- 需要补设计走查材料或截图说明；
- 需要给后续 AI / 协作者一个固定的交付描述模板。

建议一个页面或一个页面家族使用一份对照说明，不要把多个无关页面堆在同一份材料里。

## 2. 页面级对照模板

### 2.1 基本信息

- 页面名称：
- 页面路径：
- 关联模块：
- 改造目标：

示例：

- 页面名称：系统设置
- 页面路径：`frontend/src/modules/system/views/system_settings/index.tsx`
- 关联模块：`system`
- 改造目标：补齐配置差异核对、提交确认和提交成功反馈
- 参考基座：`ManagementPageHeader`、`DetailDialogWrapper`、`DetailKeyValueSection`

- 与统一设计语言不一致的地方。

## 3. 截图占位规范

如果后续补截图，建议统一使用以下占位结构。

### 3.1 页面骨架截图

- 截图名称：`<page>-overview-after`
- 建议内容：
  - 页头
  - 筛选区
  - 结果区
  - 主要操作按钮

### 3.2 详情或弹层截图

- 截图名称：`<page>-detail-after`
- 建议内容：
  - 摘要区
  - 状态标签
  - 详情分组
  - 原文 / 技术载荷区

### 3.3 结果反馈截图

- 截图名称：`<page>-feedback-after`
- 建议内容：
  - 提交成功态
  - 巡检结论
  - 下一步动作
  - 风险提示或复核入口

### 3.4 前后对照摆放建议

推荐结构：

1. 改造前首屏
2. 改造后首屏
3. 改造后详情 / 反馈
4. 文字结论

要求：

- 优先对比同一页面同一视角；
- 尽量保证截图尺寸一致；
- 截图只辅助说明，不替代文字结论；
- 截图标题直接对应页面对象，不用写模糊命名。

## 4. 可直接复用的说明模板

### 4.1 列表页

```text
改造前：页面首屏结构松散，标题区、筛选区、结果区节奏不稳定，工具动作样式不一致。
改造后：页头统一到 ManagementPageHeader，筛选与结果统一到管理页容器，状态标签与行内动作进入同一套工作台语言。
结论：页面从“普通业务页”收口为更稳定的后台管理页。
```

### 4.2 表单弹层

```text
改造前：新增 / 编辑弹层头部、滚动区和底部按钮结构不稳定，反馈主要依赖 toast。
改造后：表单容器统一到 FormDialogWrapper，结构节奏一致，提交与取消动作层级更清楚。
结论：弹层从“临时拼接对话框”收口为统一的后台工具窗。
```

### 4.3 详情页

```text
改造前：详情信息混排，摘要、状态、审计和原始数据没有清晰分组。
改造后：详情容器统一到 DetailDialogWrapper，并拆分为摘要、状态、审计和原文分组。
结论：详情页从“信息堆叠”收口为可快速扫读的只读链路。
```

### 4.4 结果反馈

```text
改造前：成功反馈主要依赖 toast，用户难以直接判断影响范围和下一步动作。
改造后：页内补齐成功摘要、巡检结论或确认态，明确影响范围、关键对象和下一步入口。
结论：页面具备了更接近专业工作台的结果反馈能力。
```

## 5. 推荐先补哪些页面

如果后续真的要做截图或前后对照，优先级建议如下：

1. `frontend/src/modules/system/views/system_settings/index.tsx`
2. `frontend/src/modules/system/views/system_monitor/index.tsx`
3. `frontend/src/modules/system/views/unified_log_management/index.tsx`
4. `frontend/src/modules/tenant/views/tenant_management/index.tsx`

原因：

- 这几页最能体现当前这轮 UI 收口的专业感；
- 同时包含首屏骨架、详情链路和结果反馈；
- 最适合做设计走查和对外交付说明。

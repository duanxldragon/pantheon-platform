# 发布验收清单

> 本文定义 Pantheon Platform 的标准发布验收口径。  
> 目标是把“代码完成”与“可合并 / 可发布”区分开，确保发布前已经完成边界确认、测试闭环和回滚准备。

## 1. 适用范围

适用于：

- 版本发布；
- 里程碑合并；
- 高风险功能上线；
- 涉及认证、租户、权限、初始化的关键变更。

## 2. 发布信息

- 发布名称：
- 目标分支：
- 发布类型：`feature / hotfix / patch / milestone`
- 发布负责人：
- 计划时间：
- 回滚负责人：

## 3. 变更范围确认

- [ ] 已明确本次包含的功能 / 修复列表
- [ ] 已明确不包含的范围
- [ ] 已确认是否涉及 `auth / tenant / system / notification / 壳层 / infra`
- [ ] 已整理高风险变更清单

## 4. 代码与结构检查

- [ ] 改动落在正确目录与分层
- [ ] 未引入第二套状态、权限、菜单、主题或认证机制
- [ ] 未保留临时开关、调试代码、一次性脚本入口
- [ ] 生成产物与生成链路保持一致

## 5. 数据与配置检查

- [ ] 已确认是否涉及主库 / 租户库变更
- [ ] 涉及 schema 变更时已验证迁移脚本
- [ ] 涉及默认数据时已验证幂等性
- [ ] 已确认配置项、环境变量、密钥与开关准备完成
- [ ] 已确认 Redis / Casbin / DB 依赖项无遗漏

## 6. 认证、租户与权限专项检查

### 6.1 认证

- [ ] 登录 / 刷新 / 退出链路正常
- [ ] 会话失效、版本刷新、强制退出逻辑正常
- [ ] 2FA / API Key 相关变更已验证

### 6.2 租户

- [ ] 租户初始化 / 状态检查链路正常
- [ ] 租户上下文不会被前端参数绕过
- [ ] Tenant DB 连接、迁移、生命周期联动正常

### 6.3 权限与菜单

- [ ] 角色、权限、菜单变更后前后端联动正常
- [ ] 菜单、标签页、动态视图挂载正常
- [ ] 后端权限兜底未缺失

## 7. 测试完成度

### 7.1 L0 静态与构建

- [ ] 前端 `type-check`
- [ ] 前端 `lint`
- [ ] 前端 `build`
- [ ] 后端编译 / 命名检查 / 基础回归

### 7.2 L1 白盒

- [ ] 关键 service / store / util / middleware 已补白盒
- [ ] 关键失败路径已覆盖

### 7.3 L2 集成

- [ ] 模块协作链路已验证
- [ ] DB / Redis / 中间件联动已验证

### 7.4 L3 E2E

- [ ] 主用户旅程已做冒烟或 E2E
- [ ] 关键异常态已做最小验证

### 7.5 L4 专项

- [ ] 安全专项已评估
- [ ] 性能专项已评估
- [ ] 未执行项与原因已记录

## 8. 文档与沟通

- [ ] 相关平台文档已更新

- [ ] PR 描述已补充风险、验证和 Review 重点
- [ ] 发布说明 / 变更摘要已准备

- [ ] 关键监控与告警项已准备

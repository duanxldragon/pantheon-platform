# 系统开发与交付检查清单

> 本文用于代码完成后的统一自查。  
> 目标是用一份短清单串起架构、编码、Review、测试和文档同步。

## 1. 架构与边界

- [ ] 已明确影响主链路：`auth / tenant / system / notification / 壳层`
- [ ] 已判断数据边界：主库或租户库
- [ ] 未引入第二套状态、权限、菜单、主题或认证机制
- [ ] 未绕开租户上下文或后端权限兜底

## 2. 实现质量

- [ ] 前端遵守 `frontend/docs/FRONTEND_CODE_STANDARDS.md`
- [ ] 后端遵守 `backend/docs/BACKEND_CODE_STANDARDS.md`
- [ ] 命名遵守前后端命名规范
- [ ] 中文、编码、换行无异常

## 3. Review

- [ ] 已按 `docs/governance/CODE_REVIEW_GUIDE.md` 自查
- [ ] 已说明风险点与未覆盖项
- [ ] 高风险动作具备确认、审计和错误反馈

## 4. 测试

- [ ] 已执行最小必要静态/构建检查
- [ ] 关键规则已补白盒或模块测试
- [ ] 主用户旅程已做冒烟或 E2E
- [ ] 涉及认证、租户、权限时已做联动验证

## 5. 文档

- [ ] 已更新对应入口文档或专题文档
- [ ] 文档术语、路径、链路与代码一致
- [ ] 未把建议误写成现状

## 6. 交付说明

提交或交付时至少说明：

- 改了什么；
- 为什么这样改；
- 做了哪些验证；
- 还剩哪些风险或后续建议。

# 前端架构现状与演进建议

> 本文记录当前前端架构的真实落地情况，并补充少量后续演进建议。  
> 它不是平台级业务规则文档，也不是组件源码翻译。

## 1. 文档定位

本文回答三件事：

- 当前前端实际上是怎么启动和分流的；
- 当前状态、菜单、标签页、动态视图如何协作；
- 哪些是已经落地的事实，哪些只是后续可选演进。

## 2. 当前技术栈

当前已落地的核心前端技术栈如下：

- `React 19`
- `TypeScript`
- `Vite`
- `Zustand`
- `Fetch API` + 自定义请求封装
- `Tailwind CSS`
- `shadcn/ui`
- `i18next`
- `Vitest`
- `Playwright`

补充说明：

- 当前仓库没有把 `React Router` 作为主导航模型；
- 当前仓库没有把 `TanStack Query` 作为通用数据层；
- 当前仓库没有把 `Zod` 作为统一表单校验基础设施。

这些能力后续可以评估是否引入，但不应在现状文档中写成既有事实。

## 3. 当前应用入口链路

前端入口主要由以下文件构成：

- `frontend/src/main.tsx`
- `frontend/src/App.tsx`

其中真正的应用分流发生在 `App.tsx`，而不是传统路由表。

当前主链路如下：

1. 挂载主题与全局能力；
2. 读取认证状态；
3. 若未登录，显示 `Login`；
4. 若已登录但租户未完成初始化，显示 `TenantSetupWizard`；
5. 若已登录且租户已就绪，进入 `MainLayout + ViewRenderer`。

这说明当前前端更接近“工作台壳层 + 状态驱动分流”，而不是“URL 驱动的多页面站点”。

---

## 4. 当前状态协作

前端当前最关键的状态源有四个：

### 4.1 `authStore`

负责：

- 登录与退出；
- 2FA 登录收尾；
- 当前用户刷新；
- 权限重载；
- refresh 成功后的统一恢复；
- 租户初始化状态检查。

它是认证、租户、系统初始化三条链路在前端的汇合点。

### 4.2 `systemStore`

负责：

- 菜单快照；
- 用户、部门、岗位、角色等系统级数据缓存；
- 系统设置与日志等系统管理数据初始化。

当前 `initialize()` 采用“大快照式加载”，适合统一重建，但也意味着首屏初始化范围偏大。

### 4.3 `uiStore`

负责：

- 标签页集合；
- 当前激活标签；
- 侧边栏折叠状态。

它承接的是“菜单 -> 标签页 -> 动态视图”的工作台交互模型。

### 4.4 动态视图管理

当前前端不是纯路由驱动，而是：

`菜单配置 -> 标签页状态 -> 视图渲染器`

这里的关键协作点包括：

- 菜单树
- `uiStore`
- `useViewManager`
- `ViewRenderer`

## 5. 当前刷新与恢复链路

权限软刷新和会话恢复并不是页面各自处理，而是统一收口在认证链路中。

当前实际流程如下：

1. 后端接口返回 `401 AUTH_VERSION_MISMATCH` 或其他需要续签的 `401`；
2. `api_client` 尝试调用 refresh；
3. refresh 成功后更新 token；
4. 执行 `authStore.refreshTenantContext()`；
5. 在该入口内依次完成：
   - `reloadAuthorization()`
   - `checkTenantStatus()`
   - `systemStore.initialize()`
6. 清理并重建标签页与菜单快照。

这样做的结果是：

- 当前用户信息可同步刷新；
- 菜单和权限可无感重建；
- 租户初始化状态不会残留旧快照；
- 工作台标签页可以在授权变化后重新收口。

## 6. 当前架构特点

### 6.1 优点

- 登录、2FA、租户初始化、正式进入系统共用一条主入口链路；
- 菜单、权限、标签页、动态视图之间的关系比较清晰；
- refresh 后的恢复动作已集中到认证链路，便于统一维护；
- 平台式工作台模型和系统管理场景较匹配。

### 6.2 当前限制

- `systemStore.initialize()` 加载范围较大，系统页与壳层初始化耦合偏紧；
- 默认标签页标题仍有硬编码文案，i18n 一致性不够完整；
- 部分架构说明文档曾混入“建议采用”的技术方案，容易与当前实现混淆。

## 7. 建议的后续演进

以下内容属于后续建议，不代表当前已经实现：

### 7.1 拆分初始化分层

可把当前系统初始化逐步拆成三层：

- 会话恢复必需数据；
- 壳层导航必需数据；
- 页面进入后按需加载的数据。

### 7.2 收紧状态边界

继续保持：

- 认证恢复由 `authStore` 收口；
- 系统快照由 `systemStore` 承接；
- 工作台交互由 `uiStore` 承接。

避免新增第二套全局状态入口。

### 7.3 补充动态视图专题

后续可以单独补一份“动态视图运行时说明”，专门解释：

- 菜单如何映射视图；
- 标签页如何恢复；
- 授权变更后如何处理不可访问页面。

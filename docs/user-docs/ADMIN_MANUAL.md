# Pantheon Platform 管理员操作手册

## 📋 文档信息

- **项目名称**: Pantheon Platform
- **文档版本**: v1.0
- **创建日期**: 2026-04-07
- **适用对象**: 系统管理员、租户管理员

## 🎯 管理员职责

### 主要职责
- 用户和权限管理
- 系统配置管理
- 监控和告警处理
- 数据备份和恢复
- 技术支持

### 权限说明
- **系统管理员**: 拥有最高权限，可以管理所有租户和系统设置
- **租户管理员**: 只能管理本租户内的用户和配置

## 👥 用户管理

### 用户生命周期管理

#### 新建用户账户

**操作步骤**:
1. 登录系统后，进入"用户管理" → "用户列表"
2. 点击"新建用户"按钮
3. 填写用户基本信息：
   ```
   用户名: jsmith (必填，4-20个字符)
   真实姓名: John Smith (必填，50字符以内)
   邮箱: john.smith@example.com (必填，唯一)
   手机号: 13800138000 (选填)
   所属部门: 技术部 (必填)
   职位: 软件工程师 (必填)
   ```
4. 设置初始密码或自动生成
5. 分配用户角色
6. 点击"保存"完成创建

**注意事项**:
- 用户名必须唯一，建议使用英文字母、数字、下划线组合
- 邮箱将用于账号激活和密码重置，确保邮箱地址正确
- 初始密码需要用户首次登录时修改

#### 批量导入用户

**操作步骤**:
1. 在用户列表页面，点击"批量导入"按钮
2. 下载Excel模板文件
3. 按模板格式填写用户信息：
   ```
   用户名* | 真实姓名* | 邮箱* | 手机号 | 部门 | 职位 | 角色
   john.doe | 约翰·多伊 | john@example.com | 138... | 技术部 | 工程师 | user
   ```
4. 上传填写好的Excel文件
5. 系统自动验证数据格式
6. 预览导入信息，确认无误后开始导入
7. 查看导入结果统计

**导入规则**:
- 用户名或邮箱重复的记录会被跳过
- 必填字段为空的记录会被跳过
- 导入失败记录会在结果中标注原因

#### 用户状态管理

**启用用户**:
1. 找到被禁用的用户
2. 点击操作列的"启用"按钮
3. 确认操作
4. 用户状态立即变更为"正常"
5. 用户可以正常登录系统

**禁用用户**:
1. 找到需要禁用的用户
2. 点击操作列的"禁用"按钮
3. 选择禁用原因：
   - 违反公司规定
   - 账户安全风险
   - 离职/转岗
   - 其他原因
4. 添加备注说明
5. 确认禁用
6. 用户立即无法登录系统

**锁定用户**:
- 发生多次密码错误尝试时自动锁定
- 管理员可以手动锁定用户
- 锁定用户需要手动解锁或等待30分钟自动解锁

## 🔐 权限管理

### 角色创建和配置

#### 创建新角色
1. 进入"角色管理"页面
2. 点击"新建角色"按钮
3. 填写角色信息：
   ```
   角色名称: 数据分析师
   角色代码: data_analyst
   角色描述: 负责数据分析和报表生成
   是否系统角色: 否
   ```
4. 点击"创建"按钮

#### 配置角色权限

**菜单权限配置**:
1. 在角色列表中，点击角色的"编辑权限"按钮
2. 在左侧菜单树中，勾选该角色可以访问的菜单
3. 系统会自动显示对应的功能权限

**操作权限配置**:
- **查看**: 可以查看列表数据
- **新增**: 可以新增数据
- **编辑**: 可以编辑数据
- **删除**: 可以删除数据
- **导出**: 可以导出数据
- **导入**: 可以导入数据

**数据权限配置**:
- **全部数据**: 可以访问所有数据
- **本部门数据**: 只能访问本部门及下级部门数据
- **仅本人数据**: 只能访问自己创建的数据

### 权限分配

#### 为用户分配角色
1. 在用户编辑页面，找到"角色分配"区域
2. 从"可用角色"列表中选择角色
3. 点击"添加"按钮
4. 角色移动到"已分配角色"列表
5. 保存用户信息

#### 为用户直接分配权限
1. 在用户编辑页面，找到"直接权限"区域
2. 点击"添加权限"按钮
3. 在权限树中勾选需要的权限
4. 点击"确定"添加
5. 直接权限会与角色权限叠加生效

### 权限继承

#### 设置父角色
1. 在创建或编辑角色时
2. 在"继承关系"区域选择父角色
3. 子角色自动获得父角色的所有权限
4. 可以在父角色基础上扩展额外权限

## 🏢 租户管理

### 租户生命周期管理

#### 租户创建流程
1. 准备租户基本信息：
   ```
   租户名称: 客户公司名称
   租户代码: client (唯一标识)
   联系人信息: 姓名、邮箱、电话
   ```
2. 配置资源配额：
   ```
   最大用户数: 根据套餐确定
   存储空间: 根据套餐确定
   API调用频率: 根据套餐确定
   ```
3. 配置数据库：
   ```
   数据库类型: MySQL
   连接池大小: 根据用户数确定
   ```
4. 提交创建申请
5. 系统自动：
   - 创建租户记录
   - 创建独立数据库
   - 初始化数据库表结构
   - 创建默认管理员账户
   - 发送账号信息到联系邮箱

#### 租户状态管理

**暂停租户**:
- 用于欠费或违规等场景
- 用户无法登录，数据保留
- 管理员仍可管理租户数据
- 恢复时需确认身份验证

**冻结租户**:
- 用于准备删除或严重违规的场景
- 所有服务停止，数据准备删除
- 冻结30天后可以永久删除
- 删除操作不可逆，需谨慎

### 租户监控

#### 查看租户健康状态
1. 进入"租户管理" → "租户监控"
2. 选择要查看的租户
3. 查看健康仪表板：
   - **健康分数**: 0-100分，90+优秀，70-90良好，<70需关注
   - **资源使用**: 实时显示资源使用情况
   - **性能指标**: 响应时间、错误率等
   - **告警信息**: 当前活跃的告警

#### 租户资源管理
1. 在租户详情页面
2. 查看资源使用情况：
   - 用户数量: 当前使用/配额
   - �储空间: 已用/总量
   - API调用: 当月使用/配额
3. 资源告警:
   - 使用率>80%: 警告告警
   - 使用率>90%: 严重告警
   - 联系租户升级套餐

## 📊 系统监控管理

### 监控仪表板解读

#### 系统概览页面

**关键指标说明**:
- **总租户数**: 当前系统中的租户总数
- **健康租户**: 系统运行正常的租户
- **警告租户**: 存在性能问题但可用
- **严重租户**: 存在严重问题需要处理

#### 性能监控指标

**响应时间**:
- **平均响应**: <300ms 为优秀
- **P95响应**: <800ms 为良好
- **P99响应**: <2000ms 为可接受

**资源使用**:
- **CPU使用率**: <70% 为健康
- **内存使用率**: <80% 为健康
- **磁盘使用率**: <85% 为健康
- **连接池使用**: <80% 为健康

### 告警处理

#### 告警级别说明
- **🔴 严重**: 立即处理，系统不可用或数据安全风险
- **🟠 警告**: 尽快处理，功能受影响或性能下降
- **🔵 信息**: 关注即可，一般性提醒

#### 告警处理流程
1. 收到告警通知
2. 登录系统查看告警详情
3. 分析告警原因：
   - 检查监控指标趋势
   - 查看相关日志
   - 分析影响范围
4. 采取处理措施：
   - **应用故障**: 重启服务、扩展实例
   - **数据库故障**: 切换主从、优化查询
   - **资源不足**: 扩容资源、清理数据
   - **安全威胁**: 隔离威胁、修复漏洞
5. 验证问题解决
6. 更新告警状态为"已处理"

## 🚨 应急处理

### 常见应急场景

#### 用户无法登录

**症状**: 用户输入正确的用户名密码，但无法登录

**处理步骤**:
1. 检查用户状态是否为"正常"
2. 检查用户是否被锁定
3. 检查用户所属租户状态
4. 重置用户密码
5. 清除浏览器缓存和Cookie

#### 系统响应缓慢

**症状**: 页面加载慢，API响应时间长

**处理步骤**:
1. 检查系统概览页面，查看性能指标
2. 检查资源使用情况
3. 查看错误率是否升高
4. 检查是否有告警未处理
5. 联系技术支持团队

#### 数据不一致

**症状**: 显示的数据与实际不符

**处理步骤**:
1. 清除浏览器缓存
2. 强制刷新页面 (Ctrl+Shift+R)
3. 检查是否有多个标签页同时操作
4. 重新登录系统
5. 如果问题持续，联系技术支持

### 备份操作

#### 日常备份
**频率**: 每日自动备份
**范围**:
- 系统配置

#### 手动备份
**适用场景**: 重大变更前
**操作步骤**:
1. 进入"系统管理" → "备份管理"
2. 点击"立即备份"按钮
3. 选择备份范围：
   - 全量备份
   - 增量备份
   - 配置备份
4. 系统执行备份操作
5. 查看备份结果

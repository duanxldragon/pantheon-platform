# 认证模块测试用例

## 📋 测试用例信息

- **模块名称**: 认证授权模块
- **测试类型**: 功能测试 + 集成测试 + 安全测试
- **测试优先级**: P0 (最高)
- **覆盖场景**: 用户注册、登录、权限、会话管理

## 🧪 功能测试用例

### 用户注册功能

#### TC-AUTH-001: 正常用户注册

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-AUTH-001 |
| **测试场景** | 用户使用有效信息完成注册 |
| **前置条件** | 系统正常运行，邮箱服务可用 |
| **测试步骤** | 1. 访问注册页面<br>2. 输入有效邮箱: test@example.com<br>3. 输入用户名: testuser<br>4. 设置密码: Secure@123<br>5. 同意服务条款<br>6. 点击"注册"按钮 |
| **预期结果** | 1. 注册成功，跳转到欢迎页面<br>2. 系统发送验证邮件<br>3. 创建用户状态为"待激活"<br>4. 返回成功提示 |
| **测试数据** | 邮箱: test@example.com<br>用户名: testuser<br>密码: Secure@123 |

| **预期结果** | 1. 显示错误提示: "邮箱已被注册"<br>2. 提示"直接登录"或"找回密码" |

### 暴力破解防护测试

#### TC-SEC-010: 密码暴力破解

| 项目 | 内容 |
|------|------|
| **用例ID** | TC-SEC-010 |
| **测试场景** | 模拟暴力破解攻击 |
| **测试步骤** | 1. 使用工具或脚本进行100次失败登录尝试<br>2. 每次使用不同的密码 |
| **预期结果** | 1. 前5次: 记录失败，允许继续<br>2. 第6次: 用户被锁定30分钟<br>3. 锁定期间无法登录<br>4. 攻击被检测和记录 |
